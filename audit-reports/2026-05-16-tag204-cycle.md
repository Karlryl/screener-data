## Tag 204a — ADR Currency Fix (closes 2 HIGH bugs)

Bug 1 (Tag 203b finding #1): `mapYahooToCanonical` was reading
`meta.reportingCurrency` from `yahoo.price.currency` (trading-quote ccy),
which is wrong for ADRs. Fixed by preferring `yahoo.price.financialCurrency`
when both fields present and different.

Bug 2 (Tag 203b finding #3): `_convertSnapshotToUSD` enumerated only
`annual.*`; `metrics.*` fields bypassed conversion silently. Fixed by
adding `CCY_DENOMINATED_METRICS` allow-list (active: revenueTTM;
reserved: fcfTTM, ebitda, enterpriseValue, bookValuePerShare,
cashPerShare).

4 smoke tests added; all 75 tests passing.

**Anchor safety:** native USD/CHF/EUR listings unchanged (the `!==`
guard means no behavior change unless financialCurrency differs from
trading-currency).

**POST-MERGE ACTION:** trigger `workflow_dispatch` on daily-pull.yml to
regenerate the snapshot corpus. Existing on-disk snapshots were written
by the pre-fix mapper and have never been FX-converted. Without refresh,
TSM/BABA fcf-yield etc. remain corrupted on disk even though new pulls
will be correct.

## Tag 204e — Winsorization Research (Tag 205 Proposal)

Phase 6 research loop. Adopted; Tag 205 implementation plan included
in report.

**Key insight:** the leverage isn't winsorization itself (well-known
stats trick); it's that median-based sector thresholds CURRENTLY AMPLIFY
ADR currency bugs from "one corrupt row" to "entire SAAS-EMEA bucket's
fcf-yield median shifts 4%→18%, silently failing legitimate stocks".

Proposed: new `methods/winsor-cap-helper.js` wired into
`sector-medians-compute.js` at 3 sites (roic, roce, fcf-yield) with
p5/p95 default band. Defense-in-depth complement to Tag 204a:

- **Tag 204a** fixes the SOURCE of bad data (ADR reportingCurrency)
- **Tag 205** (proposed) fixes the AMPLIFICATION downstream (sector
  medians robust to any single outlier)

**Anchor safety:** SAFE for all 6 anchors. Winsorization operates one
level above per-stock evaluation — trims the array fed to median(), not
the per-stock value being compared.
