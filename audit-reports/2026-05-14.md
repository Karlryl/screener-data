1. **[CRITICAL] pull-yahoo.js:442-485 + methods/sloan-ratio.js:34-55 + methods/* — Year-misaligned annual arrays silently corrupt CORE methods**
   - Root cause: `mapFTSToAnnual` pushes a `{value}` row to `annualRev` whenever `totalRevenue` is non-null, but pushes to `annualOpInc`/`annualGP`/`annualNetIncome` only when each respective field is non-null. Affects QC MUST gates (sloan-ratio, earnings-stability, margin-quality) and HYPERGROWTH classification (hypergrowth-quality-class is CORE, weight 0.15). Silent — no error, no flag, just wrong picks.
   - Fix: At the pull layer, either (a) push `null` placeholders for missing secondary fields to preserve positional alignment, or (b) emit `{year, value}` records and require methods to join by year. In each consumer method, replace the `_arrVals` (`map → filter Number.isFinite`) pattern with a single zipped per-year record list; skip the whole record when any required field is missing.
   - Compound: CG-001 (F-DP-020, F-DP-021, F-ME-003, F-ME-013, F-ME-014, F-ME-021)

   - Compound: CG-002 (F-SM-001, F-SM-002, F-SM-007, F-SM-008)

3. **[CRITICAL] methods/score-aggregator.js:146-163 — Tier scores deflate for stocks with incomputable methods, biasing against international/small-caps**
   - Root cause: `normScore = (weightedSum / totalWeight) * 100` accumulates weight in the denominator for every method including incomputable ones (which contribute 0 to numerator). A QC stock with 4 of 8 methods incomputable (typical for foreign large-cap with thin Yahoo balance-sheet coverage) is capped at score ~50 even when every computable signal passes.
   - Impact: Cohort bias: invisible push of international and small/mid-cap stocks into NEAR_MISS / REJECT regardless of underlying quality. Combined with the F-ME-003 alignment bug, alignment failures → more incomputable methods → even more deflation.
   - Fix: Normalize against only the sum of computable methods' weights: `normScore = (weightedSum / sumOfComputableWeights) * 100`. Optionally require a minimum-coverage threshold (e.g. ≥60% of weight computable) before scoring at all.
   - Compound: CG-001 (compounds with array misalignment)

4. **[HIGH] pull-yahoo.js:161-223, 616-639 — Currency invariant violation in price-only fast-path**
   - Root cause: Day-0 full pull converts marketCap to USD and sets `meta.reportingCurrency='USD'`.    - Fix: In `_priceOnlyUpdate`, look up the FX rate from `existing.meta.fxRateApplied` (or re-derive from `existing.meta.reportingCurrencyOriginal`) and apply it to `q.regularMarketPrice` and `q.marketCap` before writing. OR restrict the price-only fast-path to tickers whose `meta.reportingCurrencyOriginal === 'USD'`. Make `_convertSnapshotToUSD` pure (return a new object) and gate on a dedicated `meta.fxConverted` boolean instead of the currency string.
   - Compound: CG-003 (F-DP-007, F-DP-008, F-DQ-002, F-DQ-003, F-DQ-004)

### CRITICAL

**F-ME-003 — Systemic array-index misalignment after `_arrVals().filter(Number.isFinite)`**
- File: methods/earnings-stability.js, margin-quality.js, premium-compounder-proof.js, reinvestment-rate.js, sloan-ratio.js, hypergrowth-quality-class.js, q-spike-dataguard.js, revenue-shock-guard.js, forecast-contamination-guard.js, quarter-concentration-guard.js, deceleration-guard.js, revenue-volatility-guard.js, estimate-revision-proxy.js
- Evidence tier: T1
- Description: The local `_arrVals` pattern `arr.map(v => v == null ? null : (typeof v === 'number' ? v : v.value)).filter(v => Number.isFinite(v))` produces a compacted array that no longer aligns with the original fiscal-year index. Code iterating two parallel arrays (`revs[i]`, `opIncs[i]`) assumes same-year alignment but if any year has a missing value in either source, indices shift and year-2024 OpInc gets compared with year-2023 Revenue. Also-reported-by: data-pipeline (F-DP-020, F-DP-021 — root cause at the pull layer).
- Mechanism: annualRev=[2024:5B, 2023:4B, 2022:null, 2021:3B] and annualOpInc=[2024:1B, 2023:0.8B, 2022:0.6B, 2021:0.5B] → after `_arrVals`: revs=[5B,4B,3B] (2022 dropped), opIncs=[1B,0.8B,0.6B,0.5B] → loop `i=0..2`: pairs (5B,1B), (4B,0.8B), (3B,0.6B) — last pair is 2021-rev vs 2022-opInc. Margin wrong. Sloan denominator (assetsArr) is taken RAW (no filter), so years drift differently across the three inputs.
- Reproducer: Construct stock with `annualOpInc[1] = null` and full `annualRev`. Run margin-quality — `gmMedian` and `opMargMedian` use cross-year pairs.
- Impact: Silent false-pass/false-fail across CORE QC and HYPERGROWTH methods. Wrong picks reach production. No diagnostic indicator.
- Fix: Keep parallel arrays in original ordering with null placeholders, OR build a single zipped record list keyed by fiscal year before pairing. Replace `_arrVals` everywhere index alignment matters.
- Compound group: CG-001

**F-SM-001 — methodHistory sidecar is gitignored and ephemeral; trend signals never accumulate**
- File: detect-changes.js:44-62 + .gitignore:29-31 + .github/workflows/daily-pull.yml:111
- Evidence tier: T1
- Description: Tag 134 Phase 5.2 moved methodHistory to `external-data/method-history-state.json` to fight the 25 MB push bloat. That sidecar is gitignored. GitHub Actions runners are provisioned fresh per run (`actions/checkout@v4`), nothing in the workflow restores this sidecar (no `actions/cache`, no upload/download-artifact). `_loadMethodHistory()` returns `{}` on every run, `saveState()` writes a sidecar that is discarded at job tear-down. Also-reported-by: state-management (also F-SM-007 as the migration-path root-cause).

- Fix: Option (c) — move sidecar back into git but rely on the existing `HISTORY_WINDOW=8` cap to bound size at ~3 MB. Or add `actions/cache@v4` keyed by repository to the workflow.
- Compound group: CG-002

**F-ME-023 — score-aggregator deflates score for stocks with incomputable methods (cohort bias)**
- File: methods/score-aggregator.js:146-163
- Evidence tier: T1
- Description: `totalWeight` accumulates across all weighted methods including those with incomputable results (which contribute s=0 to numerator). `normScore = (weightedSum/totalWeight)*100`. An incomputable method contributes weight to denominator but 0 to numerator, deflating the final score artificially. Combined with F-ME-003 (misalignment causes more methods to fail / become incomputable), this is a hidden multiplier.
- Mechanism: QC: 8 methods, weights sum to 1.0. 4 incomputable (e.g. small-cap with no balance sheet) → combined weight 0.5 contribute 0 score. Remaining 4 all pass → weightedSum ≤ 0.5. `normScore = 0.5/1.0*100 = 50` → NEAR_MISS, not A-tier — even with no failures.
- Reproducer: Stock missing `annualOpInc` → quality-compounder-roic, margin-quality, earnings-stability, reinvestment-rate all incomputable. Remaining 4 methods all pass. Score ≈ 25 = REJECT despite zero failures.
- Impact: Silent push of small / international stocks (partial Yahoo data) into NEAR_MISS or REJECT. Invisible cohort bias.
- Fix: Normalize against only the sum of computable methods' weights: `normScore = (weightedSum / sumOfComputableWeights) * 100`. Optionally enforce minimum-coverage threshold (e.g. ≥60% computable weight) before scoring.
- Compound group: CG-001

## Compound Bug Groups

### Group CG-001: Year-Alignment Corruption Cascade
- Findings: F-DP-020, F-DP-021, F-ME-003, F-ME-013, F-ME-014, F-ME-021, F-ME-023
- Interaction: pull-yahoo.js `mapFTSToAnnual` (F-DP-021) and `_ftsExtractByYear` (F-DP-020) emit annual arrays where `annualRev` is full-length but `annualOpInc`/`annualGP`/`annualNetIncome` are shorter (because they're pushed only when non-null). Downstream methods (F-ME-003, F-ME-013, F-ME-014, F-ME-021) use shared index `i` across these differently-sized arrays, computing cross-year ratios silently. F-ME-023 amplifies this: misalignment → some method evaluations fail computability check → score-aggregator divides by full weight including incomputable methods → score deflated.
- Combined impact: A single Yahoo missing-field event for one fiscal year produces (a) wrong margin/Sloan/quality numbers in 7+ methods, (b) some of those methods reporting `computable=false`, (c) overall tier score deflated by the F-ME-023 normalization bug. The "bad data" is invisible — no error fires, scores just drift wrong in a way that disproportionately hurts foreign and small-cap stocks (most likely to have partial Yahoo coverage). Fix the root cause once at `mapFTSToAnnual` (push null placeholders or year-tagged records); fix at consumers as defense in depth.

### Group CG-002: Trend Signals Permanently Dead
- Findings: F-SM-001, F-SM-002, F-SM-007, F-SM-008
- Interaction: methodHistory sidecar is gitignored (F-SM-001), runners are fresh, so cold start → empty history every run. F-SM-007 is the precise state-machine root cause: the migration if-branch executes the `{}` fallback every time because both the inline path (workflow strips it) and the sidecar path (gitignored + ephemeral) are simultaneously empty. F-SM-002 means even if a runner were stable, mid-write death would silently rewind history. F-SM-008 means a sidecar write failure leaves committed methodState advanced but methodHistory frozen — silent skew between two state stores.
- Combined impact: `Trend.computeTrend()` always returns `n/a`. Method-Drift dashboards, methodology-report trend section, walk-forward trend annotations all permanently empty. The "trend" feature shipped in Tag 134 has effectively never worked in production.

### Group CG-003: Currency Invariant Decay
- Findings: F-DP-007, F-DP-008, F-DP-024, F-DQ-002, F-DQ-003, F-DQ-004, F-DP-009
- Interaction: F-DP-008 makes `_convertSnapshotToUSD` fragile (in-place mutation, idempotency on a single string flag). F-DP-007 actively breaks the USD invariant every 1-6 days via the price-only fast-path that writes local-currency values into a USD-claiming snapshot. F-DQ-002 / F-DP-009 means unsupported currencies are never converted in the first place but still pass the `$1B` USD floor. F-DQ-003 means correctly-supported GBp falls back to GBP when Yahoo varies the label, causing 100× overstated UK mcaps. F-DP-024 / F-DQ-004 means the 2024-vintage fallback table silently absorbs all of the above when fx-rates.json is unavailable.
- Combined impact: At any point in the universe, a meaningful fraction of stocks may have marketCap/price in the wrong unit. Sector medians get poisoned, peer comparisons are noise, $1B mcap floor admits sub-$10M micro-caps. The bug doesn't fire — it silently warps the universe composition. The fix order matters: stop the bleeding first (F-DP-007 price-only path), then audit existing snapshots (F-DP-008 idempotency), then add coverage (F-DP-009 / F-DQ-002).

### Group CG-005: Manifest Atomicity Cluster
