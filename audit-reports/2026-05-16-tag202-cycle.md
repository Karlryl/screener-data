## Tag 202a — pull-yahoo annualRnD backfill

**Discovery:** Bug #25 added FTS-based extraction of `annualRnD`, but
Yahoo's FTS `financials` module **omits researchAndDevelopment for
ASML, V, MA, MSFT, NVDA, GOOG** (verified by direct snapshot grep).
The legacy `quoteSummary.incomeStatementHistory.researchDevelopment`
field IS populated for those names — but `mapYahooToCanonical` was
never reading it.

**Fix:** read `researchDevelopment` from quoteSummary in
`mapYahooToCanonical`; store as primary `annualRnD`. Post-FTS merge
keeps quoteSummary RnD UNLESS FTS has strictly more non-null entries.

**Anchors unblocked on next pull (predicted):** ASML, V, MA, MSFT,
NVDA, GOOG. Resolves 3 of 4 Tag 201 QC misses (ASML/V/MA) in one fix.

**Cache discipline:** `FTS_CACHE_VERSION` stays at 2 (per CLAUDE.md
constraint). quoteSummary is fetched fresh on every pull, so the new
RnD source bypasses the FTS cache entirely.

**Side-finding (deferred):** `_convertSnapshotToUSD` may not include
`annualRnD` in its currency-conversion loop. EU/JP names could end up
with RnD-in-local mixed with Capex/OCF-in-USD. Tag 203 audit.

## Tag 202e — Score-History Design

Agent 10 produced design-only deliverable at
`audit-reports/2026-05-16-tag202-score-history-design.md`.

**Design summary:**
- Per-ticker rolling files at `score-history/<TICKER>.json`, capped 30
  entries (~150 B each)
- New `scripts/snapshot-score-history.js` runs in daily-pull workflow
  between `Generate Screener Dashboard` and `Snapshot Picks-History`
  with `AUDIT_SCORE_MULTIPLIERS=1` parity
- Append + prune in one pass; idempotent (re-run on same day replaces
  today's entry)
- O(1) read in generate-screener.js via memoized `readScoreHistory()`
- ΔScore-7d, ΔScore-30d, sparkline series exposed to dashboard
- 4147 tickers × 4.5 KB ≈ 18.7 MB (safe within 50 MB git cap)

**Not implemented in Tag 202** — flagged as Tag 203 priority. Critical
correctness invariant identified: must use same env (`AUDIT_SCORE_MULTIPLIERS=1`)
as the dashboard generation step or stored history diverges silently
from displayed scores.
