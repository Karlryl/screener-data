## New Method: roic-trend (Tag 200a)

Computes ROIC per fiscal year and reports year-over-year delta in pp.

```
roic(y) = annualNetIncome[y] / (totalAssets - totalCash - totalDebt)
value   = (roic[0] - roic[1]) * 100
pass    = value > 0
```

**Why year-over-year delta** (not 3-year trend like
gross-margin-acceleration): ROIC requires balance-sheet data, and Yahoo
provides only ~2-3 years per snapshot — 3y consecutive ROIC is too strict
in practice. The most-recent delta IS the leading-indicator signal.

A 2-year trajectory tag (`accelerating | decelerating | mixed | partial`)
is surfaced in `components.trend2y` when 3 years of balance data exist.

## New Quarantine Signature: net-income-volatility (Tag 200b)

Catches non-operating noise — impairments, write-downs, asset-sale
gains/losses, restructuring — patterns where volatility lives in
NI but not in operating income.

```
delta[i] = |NI[i] - NI[i+1]|     for i in 0..len-2
ratio    = max(delta) / annualRev[0]
pass     = ratio < 1.0
```

Gate cleanly separates the two clusters. Pattern-based, no hardcoded
tickers. Complementary to metric-divergence-guard (defense in depth):

  reporting anomaly).
- ni-volatility: NI swings cross-year (structural earnings noise).

MSTR fires both. A future bitcoin-treasury company with a recent
positive year would slip metric-divergence but be caught by ni-vol.

## Bug Fix: pbScore overflow (Tag 200c)

Self-audit caught that pbScore could exceed 100 for extreme cases:

```

  Tag 199i formula: 50.25 + 13.6 + 32.55 + 10 + 15 + 15 = 136.4
```

Cap fix: each linear component now `min(100, max(0, value))`. Max
pbScore is now strictly 100 (sum of capped weights + bonuses).

Recomputed values (relative ordering preserved):
- CRDO: 136 → 98.6
- PLTR: ~95 → ~88
- NVDA: ~57 unchanged (already below all caps)
