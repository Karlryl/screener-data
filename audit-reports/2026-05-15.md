## 🔴 REMAINING — BLOCKING (must-fix before next pull)

### F-SM-015 — alert-state.json auto-reset wipes 6200 baselines on any parse error
- file: `.github/workflows/daily-pull.yml:286-295` + `detect-changes.js:71-75`
- evidence: T1
- impact: Single byte corruption → `{}` committed → next run emits ~186 000 false METHOD_PASS_GAINED Discord events; fieldCoverage history wiped; drift detector cold-starts.
- fix: On parse error, copy aside, exit non-zero; never silently overwrite committed state.
- compound: F-SM-018, F-CI-005 (CI-005 was partially fixed in Tag 179 but the inline workflow `s = {}` fallback remains).

### F-DP-051 — Top-level fetchedAt updated when *any* currency succeeded; loadFx ignores per-currency lastSuccessAt
- file: `pull-yahoo.js:98-116`, `scripts/refresh-fx.js:99-106`
- evidence: T1
- impact: TRY/IDR/EM currency rates can be silently 30+ days stale while file appears fresh; whole EM legs mis-priced.
- fix: In `loadFx`, drop or null any currency whose `currencyMeta[c].lastSuccessAt > FX_STALE_DAYS`.
- compound: F-DQ-008, F-DP-034.

### F-EN-012 — MNDY engine-fixture expected value contradicts documented intent
- file: `engine-fixtures.js:431-466`
- evidence: T1
- impact: Test passes for the wrong contract; if engine ever correctly emits QUALIFIED for MNDY (per Fix-6 intent), the fixture fails — and if engine emits DISQUALIFIED, the documented behaviour is silently regressed. Test is anti-guardrail today.
- fix: Reconcile expected with intent (QUALIFIED + bucketAtLeast INFLECTION) OR remove the contradictory comment.
